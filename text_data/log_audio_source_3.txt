настоящее время большинство айти компаний используют систему управления проектами для обеспечения процессов разработки состояния своих продуктов данные программные системы как правило содержат различные инструменты для настройки потока задач и соответствующих бизнес-процессов
однако функция отслеживания рабочей нагрузки персонала часто не реализованы на необходимом уровне что приводит к различным проблемам которые могут стать причиной финансовых потерь
целью данной работы является разработка информационного сервиса для управления трудозатратами сотрудников айти компаний на основе данных систем управления проектами
для достижения данной цели требуется решить ряд задач а именно изучить предметную область и провести сравнительный анализ существующих систем управления проектами
формализовать требования к сервису спроектировать сервис и его реализовать

разработки по часто используют следующие инструменты
на рисунке один представлена таблица нормирование труда которая позволяет определить сотрудников превысивших оценочные трудозатраты по задачам для последующего выяснения причин отклонений
на рисунке два показана таблица установки лимита трудозатрат по сотрудникам и проектам используемая для оценки потенциальных точек перегрузки персонала и определения сотрудников выполняющих наиболее объёмную часть работ по проекту
на рисунке три изображён график сгорания задач используемый в гибких методологиях разработки и  позволяющий оценить степень выполнения задач командой по отношению к плану и прошедшему времени
на рисунке четыре представлена диаграмма плановых трудозатрат к фактическим которое позволяет оценить реальное использование ресурсов сотрудникам или командой по задачам или проектам чтобы проверить качество первоначальной оценки задач

на основе данного набора инструментов были составлены критерии сравнительного анализа для которого были выбраны наиболее используемые в индустрии системы управления проектами а именно atlassian jira easy redmine trello youtrack azure boards и asana по результатам проведённого сравнительного анализа было установлено что ни одна система
полностью не соответствует вся сформированным критериям что делает целесообразным разработку нового решения

для реализации информационного сервиса были составлены следующие функциональные требования
просмотр разницы оценочного и фактического времени выполнения задач установка плановых трудозатрат за период просмотр статистики сгорания задач подсчёт суммарных затрат за период
таким образом опираясь на представленные требования можно реализовать информационный сервис полностью соответствующий всем сформированным критериям и устраняющий рассмотренные недостатки систем управления проектами

для реализации сервиса соответствующего определённым выше требованиям при проектировании был использован клиент-серверный подход который позволяет разрабатывать обе части приложения независимо а основная нагрузка по обработке данных и системы управления проектами и подготовки отчётов обрабатывается серверным приложением снижая нагрузку на устройство клиентов приложения кроме того были составлены модель данных описывающая все необходимые сущности сервиса сформированы сценарий использования и спроектирована карта страниц клиентского приложения представлены на рисунке пять при открытии сервиса пользователь ведёт страницу авторизации с которой доступны переходы на экран выставления доступа и регистрации учётной записи после успешной авторизации пользователь перенаправляется на страницу просмотра суммарных трудозатрат за период откуда возможны переходы на экран управления планом ресурсов просмотр статистики выполнения плана и анализа фактических трудозатрат сотрудников компании

требуемый сервис был реализован с использованием технологий представленных на слайде
для разработки серверного приложения использовался язык typescript являющийся javascript совместимым языком со статической типизации позволяющий создавать более безопасные по сравнению с javascript приложения
в качестве базового фреймворка использовался nodejs экспресс а для авторизации и аутентификации пользователей библиотека паспорт js
хранения данных в системе реализовано с помощью субд mongodb и библиотеки mongoose для доступа к ним клиентское приложение было реализовано при использовании языка javascript является стандартным для браузерных приложений
а также фреймворка nextjs через основанного на reactjs
основное преимущество nextjs возможность использования серверного приложения для подготовки отображения сложных графиков и таблиц тем самым снижаем нагрузку на браузер как следствие на устройство пользователя
для создания пользовательского интерфейса была выбрана библиотека ant дизайн позволяющая использовать готовые компоненты для упрощения процесса разработки а для реализации графиков и таблиц были использованы библиотеки chartjs и pivottablejs соответственно
взаимодействия клиентского и серверного приложения было реализовано с помощью технологии graphql
использование которой позволяет клиентскому приложению запрашивать только необходимую в данный момент информацию для отображения пользователю что соответствует снижению нагрузки на сервис
кроме того graphql совместим с mongodb на уровне организации данных так как обе технологии используют документно ориентированный подход
